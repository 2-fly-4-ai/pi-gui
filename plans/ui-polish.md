# UI Polish — Minimalist Cleanup Pass

Status: complete and archived 2026-07-10. All listed batches have implementation notes and real-Electron screenshot/regression proof below; future polish findings should start a new focused plan instead of reopening this cleanup pass.

## Goal

Understated, high-leverage visual polish. No redesign, no new chrome: consistency, restraint, and small fixes that make the whole surface feel intentional. Verified against real-Electron screenshots (captured 2026-07-08 via `apps/desktop/tests/dev/ui-review-screenshots.spec.ts`).

## Constraints

- Minimalist only — nothing that adds visual weight or new UI concepts.
- Keep the Codex-style direction; light and dark must both be first-class.
- Each item verified with before/after screenshots on the real Electron surface (the capture spec above is the tool; run with `UI_REVIEW_OUT_DIR=<dir>`).
- Phase 3's CSS split has already landed; token work (T-group) should now update `styles/base.css` plus the split feature stylesheets in one mechanical pass.

## Findings & fixes

### T. Token discipline (root cause of most dark-mode roughness)

- **T1. Muted-gray sprawl.** `base.css` defines 7 near-identical muted tokens (`--muted`, `--muted-strong`, `--muted-soft`, `--muted-subtle`, `--muted-icon`, `--muted-path`, `--muted-settings`). Collapse to 3 (`--muted-strong`, `--muted`, `--muted-faint`); alias then delete.
- **T2. No scale tokens.** 20 distinct border radii, 19 font sizes (incl. 10.5/11.5/12.5px oddballs), ad-hoc z-indexes (1–200). Add `--radius-sm/md/lg/xl/full`, a type scale (11/12/13/15 + display sizes), and a named z-index ladder; migrate mechanically.
- **T3. Hardcoded hexes → tokens.** `main.css` has ~200 `:root.dark` override blocks compensating for hardcoded light colors (41× `#fff`, 26× `#dfe4ee`, transcript table `#dfe3eb`, blockquote `#d8deea`, link `#3358d4`, …). Every new token kills its override block. This is the single biggest "dark mode feels slightly off" fix.
- **T4. Motion consistency.** Transitions are an ad-hoc mix (120/140/150/160ms). One pair: `--transition-fast: 120ms ease` (hover/press) and `--transition-slow: 200ms ease` (panels).
- Implemented 2026-07-09: added shared radius, type-size, z-index, and motion primitives to `styles/base.css`, then migrated the topbar/titlebar, shortcut tooltip, GitHub quick-actions menu/dialog, and adjacent shell chrome in `styles/shell.css` to consume those tokens. This is a focused first T-section checkpoint; the transcript token sweep and full stylesheet mechanical migration remain open.

### C. Transcript & markdown (highest-visibility surface)

- **C1. Tables.** Headers render browser-default bold+centered; cells have full 1px grid borders (hardcoded `#dfe3eb`) that read harsh in dark. Fix: left-align `th`, horizontal row rules only (`border-bottom: 1px solid var(--line)`), subtle `th` background or just weight-600, no vertical lines.
- **C2. Links.** Hardcoded `#3358d4`, no dark variant, no hover affordance. Token + underline on hover.
- **C3. Dark code blocks.** Near-black syntax card nested inside a lighter bordered wrapper produces a double-frame. One surface, one border, one radius.
- Implemented 2026-07-09: transcript markdown tables now use left-aligned headers, tokenized header surfaces, horizontal row rules only, and no vertical grid borders. Markdown links use a new light/dark `--link` token with an underline affordance. Inline code, fenced code blocks, and blockquotes now use app surface/line/text tokens, and the old dark-mode near-black code-block override was removed so dark mode keeps one tokenized code surface instead of a double-frame. Added Electron coverage in `timeline-layout.spec.ts` that resolves CSS variables on the real renderer surface and verifies light/dark link/code/table/blockquote styling.
- **C4. Sidebar previews show raw markdown** ("## Plan I looked at \`src/…"). Strip markdown syntax when deriving preview text (plain-text render of first line).
- Implemented 2026-07-09: session/sidebar previews now run through a shared plain-text formatter that removes common markdown markers, links, inline code, checklist/list syntax, HTML tags, and empty fence lines before publishing preview text. Added unit coverage for the formatter and `previewFromTranscript`, plus Electron coverage that verifies markdown assistant output renders as plain sidebar preview text.
- **C5. Display-mode tiles render full-size markdown headings** ("Plan" as h2 inside a small tile). Scale transcript typography down inside tile context (container class already exists to hook).
- Implemented 2026-07-09: Display Mode tile transcript markdown now scopes headings to compact tile typography, tightens markdown gap/list/code/table spacing inside `.display-mode-tile__transcript`, and keeps the full thread transcript typography unchanged. Added Electron coverage that renders a `##` heading in a Display Mode tile and verifies the h2 is compact with zero browser-default margins.

- **A1.** `--accent` (#6a55f2) exists but appears almost nowhere (~39 uses, mostly focus rings); primary buttons are near-black/white. The neutral look is good — make it a decision: keep neutral buttons, but use the accent consistently for exactly one thing (focus rings + selected/pinned states, as display-mode already does). Remove stray accent one-offs that don't fit that rule.
- Implemented 2026-07-09 focused audit slice: removed stray accent usage from link-like actions and neutral hover chrome. Model onboarding and skill-profile management actions now use `--link`, the model selector search border is neutral until focus/state styling applies, Display Mode side-panel resize handles use a neutral hover fill, and old hardcoded `var(--accent, #...)` purple fallbacks were removed. Remaining A1 work is a broader semantic review of status/activity accent usage after the token sweep.
