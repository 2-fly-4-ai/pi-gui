# Phase 3 — Renderer Decomposition (App.tsx, CSS, src layout)

## Goal

Break the renderer monoliths into feature-scoped modules so phase 4 (state sync) and all future feature work stop funneling through one 4,073-line file.

## Findings being fixed

- `App.tsx`: 4,106 lines, 86+ hook calls, one component owning composer, dialogs, palette, git actions, panels, settings routing, keyboard handling.
- `src/styles/main.css`: 8,633 lines in one file.
- `src/`: 93 top-level files; feature grouping already implied by names but not by folders.
- Preload/main boilerplate partially addressed in phase 1 (guarded IPC wrappers); renderer side of that symmetry lands here.

## Constraints

- Behavior-preserving refactor: no UX changes in this phase. Transcript/timeline behavior and session correctness are product features — regressions there are release blockers, not polish.
- Prefer clean extraction over wrapper indirection; if a boundary fights back, reimplement the slice cleanly rather than threading 20 props through.
- Phase 2's lint + unit layer must be green before starting; e2e core suite is the regression harness throughout.
- Incremental checkpoints: each extraction is a separate commit that builds and passes core e2e.

## Steps

1. **Map the state.** Inventory App.tsx's hooks into domains (expected: composer, panels/layout, git actions + dialogs, command palette + shortcuts, settings/session selection, subagents/observability, notifications/onboarding). Write the map into this file before cutting, including the intended extraction order and the cross-domain state each slice still needs.
2. **Extract per-domain hooks** (`useComposerState`, `useGitActions`, `usePanelLayout`, `useCommandPalette`, `useSessionSelection`, ...) into `src/features/<domain>/`. Each hook owns its state + IPC calls; App.tsx becomes composition + layout.
3. **Introduce a lightweight shared store** for the cross-cutting pieces (DesktopAppState snapshot, selected transcript): a small context/store module (`src/state/`) wrapping `piApp.getState`/`onStateChanged`, with selector-based subscription so components re-render on their slice only. This is the seam phase 4 plugs into.
4. **Folderize `src/`** by feature: `composer/`, `timeline/`, `settings/`, `git/`, `terminal/`, `panels/`, `sidebar/`, `state/`, `shared/`. Mechanical moves with import updates; one commit.
5. **Split `main.css`** along the same feature folders (colocated `<feature>.css` imported by the feature entry, or CSS modules if the diff stays reviewable — decide on the first extraction and stick with it). Keep `base.css` as the tokens layer; dedupe obvious repetition during the split but no visual redesign.
6. **Target sizes.** App.tsx < 400 lines; no renderer file > 800 lines except generated/icon files; no CSS file > 1,500 lines.
